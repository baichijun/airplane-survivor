已重构虚拟摇杆方案，修复你提到的两个问题，并做了操控优化。

## 问题原因

**1. 界面切换后瞬移 / 加速**
- 暂停时曾用 `touchTarget` 把飞机 **拉向手指坐标**；手指在左下摇杆区域时，目标点就是摇杆附近，看起来像飞机「吸」到摇杆位置
- 选完升级后又 **强制还原** 到进入暂停前的坐标，产生明显回弹

**2. 左下/右下与左上/右上速度不一致**
- 原先 8 方向吸附 + 斜向 `0.45` 倍速，斜向（含左下、右下、左上、右上）比直行慢很多

## 改动摘要

| 项目 | 调整 |
|------|------|
| 移除 `touchTarget` | 暂停/战斗都只靠摇杆位移，不再把飞机拽向触摸点 |
| 取消暂停位置还原 | 选完奖励后飞机停在当前位置，不再瞬间跳回 |
| 场景切换清输入 | 进入/离开暂停时 `clearMobileInput()`，避免残留拖拽 |
| 模拟量方向 | 取消 8 向吸附，用归一化向量，**各方向同速**（与键盘一致） |
| 浮动摇杆 | 在屏幕左下约 52% 宽 × 下方区域按下时，底盘出现在拇指下，跟手更好按 |
| 移除斜向减速 | 删除 `JOYSTICK_DIAGONAL_SPEED_MULT` |

激活阈值仍为约 30% 行程，未推够时飞机静止。
