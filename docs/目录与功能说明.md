# 目录与功能说明

本文档说明项目每个目录/文件的职责，以及**想调整某功能时应去改哪里**。维护代码或和 AI 协作时，可直接引用文件名。

---

## 一、整体架构

```
浏览器加载 index.html
    └── main.ts 创建 Game
            ├── Engine      画布尺寸、缩放、清屏
            ├── Input       键盘 / 触摸 / 鼠标
            ├── 状态机       MENU → PLAYING ⇄ PAUSED → GAME_OVER
            ├── entities    玩家、敌机、子弹（数据 + 绘制）
            ├── systems     战斗、碰撞、刷怪、经验（纯逻辑）
            ├── config      数值与升级表（改平衡主要在这里）
            └── ui          HUD 与各类 overlay 绘制
```

**数据流（每帧）**：`Input` → `Player` 移动 → `CombatSystem` 开火 → `SpawnSystem` 刷怪 → 实体 `update` → `CollisionSystem` 结算 → `ExpSystem` 经验 → 渲染 `draw`。

## 三、`src/` 源码目录

### 入口与样式

| 文件 | 作用 |
|------|------|
| `src/main.ts` | 程序唯一入口：`new Game('game-canvas')` |
| `src/style.css` | 全屏黑底、禁止滚动与文本选中、`#game-canvas` 铺满视口 |

### `src/types/` — 类型定义

| 文件 | 作用 |
|------|------|
| `types/index.ts` | 全局类型：`GameState`、敌机类型 `EnemyType`、升级 ID `UpgradeId`、`EnemyConfig`、`UpgradeDef` 等 |

**何时改这里**：新增游戏状态、新敌机种类、新升级类型时，先在此扩展类型，再改 config / systems。

---

### `src/config/` — 数值与配置（改平衡的首选位置）

| 文件 | 作用 |
|------|------|
| `config/balance.ts` | **核心数值表**：画布尺寸、玩家初始属性、敌机三种配置、刷怪间隔、经验公式、无敌时间、接触伤害等 |
| `config/upgrades.ts` | **升级奖励池**：名称、描述、随机抽取逻辑、`applyUpgrade()` 各奖励的实际数值效果 |

---

### `src/core/` — 引擎与主控

| 文件 | 作用 |
|------|------|
| `core/Engine.ts` | Canvas 初始化、高清屏适配、逻辑坐标与屏幕坐标转换、帧间隔上限、清屏 |
| `core/Input.ts` | WASD/方向键、触摸拖拽、鼠标按下/移动；提供移动方向与点击坐标 |
| `core/Game.ts` | **游戏大脑**：状态机、主循环 `loop`、整合所有 system/entity/ui、点击菜单/升级/重开 |

---

### `src/entities/` — 游戏对象（属性 + 更新 + 绘制）

| 文件 | 作用 |
|------|------|
| `entities/Player.ts` | 玩家位置、生命值、攻速/弹道/移速等属性、受伤无敌帧、移动与三角形绘制 |
| `entities/Enemy.ts` | 敌机属性、向下移动、瞄准玩家、血条、矩形绘制 |
| `entities/Bullet.ts` | 子弹速度、伤害、阵营（玩家/敌机）、飞出屏幕后销毁、圆形绘制 |

---

### `src/systems/` — 游戏逻辑（无 UI，可被 Game 调用）

| 文件 | 作用 |
|------|------|
| `systems/CombatSystem.ts` | 玩家多弹道散射、敌机朝玩家方向射击 |
| `systems/CollisionSystem.ts` | 子弹↔敌机、子弹↔玩家、敌机接触玩家；返回击杀列表与是否命中玩家 |
| `systems/SpawnSystem.ts` | 按时间刷怪、间隔随时间缩短、按游戏时长解锁 fighter/tank 类型 |
| `systems/ExpSystem.ts` | 被动经验、击杀加经验、升级判定与扣经验 |

---

### `src/ui/` — 界面绘制（Canvas 上画文字和面板）

| 文件 | 作用 |
|------|------|
| `ui/HUD.ts` | 游戏中顶部：时间、击杀数、等级、生命值条、经验条 |
| `ui/LevelUpOverlay.ts` | 升级暂停时的三选一卡片布局、点击检测 |
| `ui/GameOverOverlay.ts` | `GameOverOverlay`：结束统计与「重新开始」；`MenuOverlay`：标题、操作说明、「开始游戏」 |

---

### `src/assets/` — 静态资源

| 文件 | 作用 |
|------|------|
| `assets/vite.svg` | Vite 脚手架遗留图标，**当前游戏未引用** |
| `assets/typescript.svg` | 同上，**未引用** |

## 五、关键流程说明

### 1. 游戏启动

`main.ts` → `Game` 构造函数 → 创建 `Engine`、`Input` → `requestAnimationFrame` 进入 `loop`。

### 2. 主循环（`Game.loop`）

1. 计算 `dt`（经 `Engine.clampDelta` 限制）
2. `handlePointer()` — 处理菜单/升级/重开点击
3. `update(dt)` — 仅在 `PLAYING` 时更新实体与系统
4. `render()` — 清屏、画背景、实体、HUD、overlay

### 3. 升级流程

经验满 → `Game.triggerLevelUp()` → 状态变为 `PAUSED` → `LevelUpOverlay` 显示 3 选项 → 点击后 `applyUpgrade()` → 回到 `PLAYING`（若仍满经验则连续升级）。

### 4. 游戏结束

玩家 `hp <= 0` → 状态 `GAME_OVER` → 显示统计；点「重新开始」回到 `MENU`。

## 六、修改建议

1. **改数值优先看 `config/balance.ts` 和 `config/upgrades.ts`**，避免在 entity 里写魔法数字。
2. **改玩法逻辑看 `systems/`**，**改表现看 `entities/` 的 `draw()` 和 `ui/`**。
3. **跨模块改动时以 `core/Game.ts` 为调度中心**，确认状态机是否需新状态。
4. 改完执行 `npm run build` 确认类型检查通过；开发时用 `npm run dev` 实时预览。
