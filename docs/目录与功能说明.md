# 目录与功能说明

本文档说明项目每个目录/文件的职责，以及**想调整某功能时应去改哪里**。维护代码或和 AI 协作时，可直接引用文件名。

---

## 一、整体架构

```
浏览器加载 index.html
    └── main.ts 创建 Game
            ├── Engine      画布尺寸、缩放、清屏
            ├── Input       键盘 / 触摸 / 鼠标 / 虚拟方向键
            ├── 状态机       MENU → PLAYING ⇄ PAUSED → GAME_OVER（含标准/简单模式）
            ├── entities    玩家、无人机、敌机、Boss、子弹
            ├── systems     战斗、碰撞、刷怪、Boss、经验
            ├── config      数值、升级表、宝物表
            └── ui          HUD、overlay、舰体绘制、移动端控件
```

**数据流（每帧）**：`Input` → `Player` 移动 → `CombatSystem` 开火（自机 + 无人机 + 敌机 + Boss）→ `SpawnSystem` / `BossSystem` 刷怪 → 实体 `update` → `CollisionSystem` 结算 → `ExpSystem` 经验 → 渲染 `draw`。

**暂停类型**（`PauseKind`）：
- `level_up`：升级三选一
- `relic_reward`：Boss 击杀后宝物三选一（含跳过）

### `src/config/` — 数值与配置（改平衡的首选位置）

| 文件 | 作用 |
|------|------|
| `config/balance.ts` | **核心数值表**：画布、玩家初始属性、敌机配置、刷怪、激光时序、Boss、宝物概率、无人机基础倍率、升级倍率、**简单模式**等 |
| `config/upgrades.ts` | **升级奖励池**（7 项）与 `applyUpgrade()` |
| `config/upgradeDisplay.ts` | 升级界面属性前后对比文案 |
| `config/relics.ts` | **宝物池**（8 种）、`applyRelic()`、跳过选项、运行时查询（护盾冷却、经验倍率、无人机追踪等） |

#### 当前升级选项（`UpgradeId`）

| ID | 名称 | 效果 |
|----|------|------|
| `attackSpeed` | 急速射击 | 自机射击间隔 ×0.85（+15% 攻速） |
| `bulletDamage` | 火力强化 | 自机子弹伤害 +1 |
| `bulletCount` | 多重弹道 | 弹道数 +1 |
| `moveSpeed` | 引擎升级 | 移速 +10% |
| `maxHp` | 装甲强化 | 最大生命 +1 并恢复（受加固舰体倍率影响） |
| `droneDamage` | 无人机火力 | 无人机伤害 ×1.5（可叠加） |
| `droneAttackSpeed` | 无人机速射 | 无人机攻速 ×1.3（可叠加） |

> 每次**角色升级**（含 Boss 击杀升级）还会自动 +1 最大生命并恢复 1 生命（`LEVEL_UP_MAX_HP_BONUS` / `LEVEL_UP_HEAL_BONUS`，受加固舰体倍率影响）。

> 已移除 `bulletSpeed`（穿甲弹）。自机弹速仍由 `PLAYER_INITIAL.bulletSpeed` 决定，暂无升级项。

#### 当前宝物（`RelicId`）

| ID | 名称 | 效果概要 |
|----|------|----------|
| `passiveShield` | 被动护盾 | 护盾冷却翻倍；冷却完毕时受击自动开盾 |
| `droneTracking` | 无人机追踪 | 无人机子弹追踪最近敌机 |
| `expAmplifier` | 经验增幅 | 所有经验 +40% |
| `vampiricRounds` | 吸血弹丸 | 击杀敌机回复 1 生命（5 秒 CD） |
| `ricochetShots` | 弹射攻击 | 击中敌机**先造成伤害**，30% 弹射向另一敌机并可再次伤害 |
| `salvageBoost` | 回收引擎 | 受击 33% 免伤 |
| `fortifiedHull` | 加固舰体 | 最大生命翻倍；之后生命上限与恢复效果翻倍 |
| `kineticConversion` | 动能转化 | 每击中敌机 8 次，主动护盾冷却 -1 秒 |

跳过选项「强化舰体」：最大生命 +30%，恢复 50% 最大生命（受加固舰体倍率影响）。

### `src/systems/` — 游戏逻辑

| 文件 | 作用 |
|------|------|
| `systems/CombatSystem.ts` | 自机多弹道、无人机、敌机射击、Boss 四种攻击模式 |
| `systems/CollisionSystem.ts` | 子弹碰撞、接触伤害、弹射逻辑、激光线段判定 |
| `systems/SpawnSystem.ts` | 敌机刷怪、间隔衰减、fighter/tank 解锁 |
| `systems/BossSystem.ts` | Boss 生成调度、已击败 Boss 计数（敌机生命缩放） |
| `systems/ExpSystem.ts` | 被动经验、升级判定 |

#### Boss 攻击模式（`CombatSystem.fireBoss`）

| 模式 | 行为 |
|------|------|
| 0 | 瞄准散弹（发数随 Boss 序号增加，夹角见 `BOSS_SCATTER_ANGLE`） |
| 1 | 多道激光（第 n 只 Boss 同时 n 道，起点沿舰宽分布） |
| 2 | 环形全向弹幕 |
| 3 | 垂直长弹 + 侧翼圆弹 |

#### 激光时序（敌机 tank / Boss）

预瞄 0.3s → 间隔 0.3s → 攻击 0.3s（常量见 `balance.ts`）。

**激光伤害**：同一帧内多道激光重叠只扣 1 次生命；每道光束接触后立即标记 `laserHitApplied`，无敌/闪避时也不会重复判定。

#### 弹射攻击流程（`CollisionSystem`）

1. 子弹命中敌机 → **立即 `takeDamage`**
2. 若拥有 `ricochetShots` 且随机 < 30%，且存在其他未命中过的敌机 → 转向并保留子弹
3. 弹射后子弹可继续命中下一架敌机并再次造成伤害
4. 同一子弹不会对同一敌机重复造成伤害（`bullet.hitEnemies`）

## 五、关键流程说明

### 1. 游戏启动

`main.ts` → `Game` 构造函数 → 创建 `Engine`、`Input` → `requestAnimationFrame` 进入 `loop`。

主菜单提供 **标准模式** 与 **简单模式**：
- **标准模式**：初始生命见 `PLAYER_INITIAL.maxHp`（默认 8）
- **简单模式**：初始生命为标准 × `EASY_MODE_INITIAL_HP_MULT`（默认 ×3）；战斗中每 `EASY_MODE_REGEN_INTERVAL` 秒恢复 `EASY_MODE_REGEN_AMOUNT` 点生命（未满时）

### 2. 主循环

1. 计算 `dt`（`Engine.clampDelta` 限制）
2. `handlePointer()` — 菜单/升级/宝物/重开点击
3. `update(dt)` — `PLAYING` 时更新实体与系统
4. `render()` — 背景、实体、HUD、overlay

### 3. 升级流程

经验满 → `triggerLevelUp()` → `PAUSED`（`level_up`）→ 三选一 → `applyUpgrade()` → 若仍满经验则连续升级。

### 4. Boss 与宝物

首个 Boss 30s 出现 → 击杀 → 补满本级经验并升级 → `PAUSED`（`relic_reward`）→ 宝物或跳过 → 后续 Boss 间隔 30s，敌机生命随已击败 Boss 数 ×1.2 累积。

**Boss 战刷怪**：Boss 存活时仍刷新敌机；第 n 波 Boss 同屏最多 n 架敌机（`getEnemyCap`），无 Boss 时为 `MAX_ENEMIES`（20）。

### 5. 游戏结束

`hp <= 0` → `GAME_OVER` → 点「重新开始」回 `MENU`。

## 六、修改建议

1. **改数值优先看 `config/balance.ts`**，升级/宝物看 `upgrades.ts` / `relics.ts`。
2. **改玩法逻辑看 `systems/`**，改表现看 `entities/draw` 与 `ui/`。
3. **跨模块以 `core/Game.ts` 为调度中心**。
4. 改完执行 `npm run build`；开发用 `npm run dev`。
